Introduction

  • This note replicates Bird, Karolyi, and Ruchti Understanding the numbers game (BKR; 2019, JAE).

Simulating the data

  • Since I dont have access to the original data, I simulate the observed data using the estimated parameters in BKR.

Parameters

  • \(\theta = (\eta, \gamma, \psi^2, \zeta)\)
set.seed(2020)

# The estimated values that apperar in Table 4
eta <- 0.5  # marginal cost
gamma <- 2.0817    # curvature
psi2 <- 0.8201   # variance
zeta <- 3.7105    # variance multiplier
theta <- c(eta, gamma, psi2, zeta)

Benefit function

  • Since the estimates of full model (coefficients on polynomials) are not reported, I approximate the capital market benefit function by compute_benefit.R.

Latent earnings distribution

  • I assume that latent earnings distribution is generated by normal distribution.
  • I set the number of firms (observations) to \(100,000\).
    • In BKR, the number of observations is \(49,604\).
  • I use the following multinomial distribution as a discretized normal:

\[ e \sim \textit{Multi}(N, P)\\ \mathcal{X} = \{ -20,-19,\dots,0,\dots,19,20 \} \\ P = (\phi(-20),\dots,\phi(20))) \]

  • where
    • \(\phi(\cdot)\) is the pdf of normal distribution with mean zero and variance \(\sigma\).
    • I set \(\sigma = 0.5\).
    • Each elemnt of \(P\) is normalized so that \(\sum_{b=-20}^{20}\phi(b)\) = 1$.
# number of firms
N <- 100000
sigma <- 5

df_firm <- simulate_firm(S = N, sigma)

Manipulation

  • Each firm decides the amount of manipulation, \(m\), according to the maximization problem.
  • compute_utility computes the following utility: \[ u_{\text {Discrete}}(b, m, \theta)=\sum_{-20}^{20} \phi_{m, \theta}(\varepsilon) \mathcal{B}(b+m+\varepsilon) d \varepsilon-\beta_{b} m^{\gamma} \]
  • Since I dont have the original data, I am assuming the following benefit function.
    • There is a jump by \(1.5\) at \(R=0\).

\[ \mathcal{B}(R)=\begin{cases} 0.05R-1 & R<0\\ 0.05R+0.5 & R\ge0 \end{cases} \] - compute_optimal_manipulation_simulated.R randomly draws \(\beta\) for each firm and computes \(m^*\) for each firm.

Step 1. Candidate \(\theta\)

  • I set the initial \(\theta\) to be the true \(\theta\).
theta_init <- theta

Step 2. Simulate optimal firm behavior

  • I compute the transition matrix, \(P\).
  • Considering the computational cost, I set the number of simulations \(S\) to be \(1,000\).
    • The original paper uses \(S=10,000\).
  • compute_P.R computes the matrix \(P(\theta)\) that maps latent earnings to reported earnings.
    • compute_P.R internally calls compute_optimal_manipulation_for_e.R, which computes the empirical probability of manipulation for each bin.
    • compute_P() uses parallel computation by default. If you prefer an usual sequential computation, set parallel = FALSE.
S <- 1000   # number of simulations

P <- compute_P(theta_init, S)

Step 3. Recover the latent earnigs distribution

  • I recover the latent earnings distribution by inverting \(P\).
  • Since some bins do not contain any firms, I have to make rows for such bins.
  • Observed bins that are outside \([-20,20]\) is excluded from the data.

Step 4 Optimizetion

  • Lastly, using SMM, I estimate the parameter.

    • This estimation method might be problematic since the moment condition is not grounded on theory.

  • The objective function is \[ L(\theta) = \left[\left(\begin{array}{l} \pi\\ \boldsymbol{x}_{S}(\theta)_{(2,\ldots T)} \end{array}\right)-\left(\begin{array}{l} \boldsymbol{x}_{S}(\theta)\\ \boldsymbol{x}_{S}(\theta)_{(1,\ldots T-1)} \end{array}\right)\right]^{\prime}\Omega\left[\left(\begin{array}{l} \pi\\ \boldsymbol{x}_{S}(\theta)_{(2,\ldots T)} \end{array}\right)-\left(\begin{array}{l} \boldsymbol{x}_{S}(\theta)\\ \boldsymbol{x}_{S}(\theta)_{(1,\ldots T-1)} \end{array}\right)\right] \]

  • GMM_objective.R computes the value of objective function for a candidate set of parameters.

    • This function proceeds as follows:
    • Compute \(P\) by compute_P.R.
    • Compute \(\pi\) and \(x\) by compute_x.R and compute_pi.R
    • Compute the objective funciton.
GMM_objective(theta_init, df_observed)

Original simulated obj

  • Alternatively, as BKR briefly mentioned, we can impose a parametric assumption on the latent distribution.
  • CMD_obj.R computes the squared some of difference between inverted probability and the normal probablity for each bin.
    • The number of simulation \(S\) should be at least \(100,000\) to get reasonable result.
    • We need to specify the variance of the latent distribution, which I assume to be \(5\).
  • Since \(\zeta\) seems not to be identified, I fixed \(\zeta = 3.71\), the true value.
  • I focus on \eta in this optimization.
# simulate data
S <- 100000
df_simulated <- simulate_firm(sigma = 5, S)

# optimization
optim_result <- optimx::optimx(par = 0.3,
                               theta = c(gamma, psi2, zeta),
                               fn = CMD_obj_eta,
                               df_observed = df_observed,
                               df_simulated = df_simulated,
                               S = S,
                               method = 'BFGS')

Counterfacturals

  • See what happens if we increase the marginal cost parameter, \(\eta\).
    • Since manipulation becomes more costly, firms that find manipulation beneficial are limited to just below the zero threshold.
theta_new <- theta
theta_new[1] <- 2     # increase the marginal cost to one

df_firm_utility_new <- compute_optimal_manipulation_simulated(df_firm, theta = theta_new, S = N)
plot_latent_reported(df_firm_utility_new)
